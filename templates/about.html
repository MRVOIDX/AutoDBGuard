About AutoDBGuard

A student-built system for safe, AI-assisted database queries.

📄 Project Description

AutoDBGuard is a risk-aware SQL execution system that bridges the gap between natural language and safe database access. It was built to demonstrate how large language models can generate SQL from English descriptions — and how those queries can be automatically validated and made safe before execution.

The system implements a multi-agent pipeline — each function acting as a specialized “agent” — to progressively analyse, score, and enforce safety policies on every generated query. This architecture mirrors real-world AI safety workflows in a simple, educational format.

Whether you're a student learning about SQL, AI, or database security, AutoDBGuard shows the full journey from a human question to a safely executed database result.

🏗 Architecture — Agent Functions

Each “agent” in AutoDBGuard is a plain Python function. There are no microservices, no message queues, no containers — just clean, composable functions called in sequence by the orchestrator.

orchestrator()

Coordinates all agents and returns a unified result to the frontend.

generate_sql_with_groq()

Sends the user's natural language to Groq and extracts the SQL response.

analyze_structure()

Inspects the SQL text for dangerous keywords and missing safety clauses.

analyze_execution_plan()

Runs EXPLAIN QUERY PLAN to detect full table scans at zero cost.

calculate_risk()

Accumulates a numeric risk score and maps it to a named risk level.

enforce_policy()

Blocks, revises, or clears the query for execution based on risk level.
